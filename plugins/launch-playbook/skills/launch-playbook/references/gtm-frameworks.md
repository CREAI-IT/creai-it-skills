# GTM Frameworks & Strategic Tools

## Positioning Framework: April Dunford's "Obviously Awesome"

Apply these 5 components in exact order:

### Step 1: Competitive Alternatives
**Question**: "What would your best customers use if you didn't exist?"

Common alternatives:
- Spreadsheets / manual processes
- Hiring a freelancer or intern
- Competitor products
- Doing nothing / living with the problem

**Why this comes first**: Your product's value only makes sense relative to what it replaces.

### Step 2: Unique Attributes
**Question**: "What features/capabilities do you have that alternatives don't?"

Rules:
- Must be factual, verifiable capabilities (not marketing claims)
- "Better UX" is not a unique attribute — it's an opinion
- "We process 10x faster because of [specific architecture]" is a unique attribute

### Step 3: Value (and Proof)
**Question**: "What benefit do those unique attributes enable? What's the proof?"

Translate each attribute to a benefit:
```
Attribute: "Real-time collaborative editing"
Benefit:   "Kill the version-control chaos that delays every project"
Proof:     "Teams save 12 hours/week on average (based on 200-user survey)"
```

### Step 4: Target Market Characteristics
**Question**: "Who cares most about that value? What makes them care?"

Segment by **pain intensity**, not demographics:
```
Bad:  "Small businesses" (too broad)
Good: "Marketing agencies with 5-20 employees who manage 10+ client accounts
       simultaneously and lose hours switching between tools"
```

### Step 5: Market Category
**Question**: "What market context makes your value obvious?"

Three positioning styles:

| Style | When to Use | Template |
|---|---|---|
| **Head-to-Head** | Strong competitor, you beat them on a key dimension | "Like [competitor] but [better at X]" |
| **Big Fish, Small Pond** | You dominate a subsegment | "The [category] built for [specific segment]" |
| **Category Creation** | Fundamentally new approach (rare, expensive) | "The first [new category name]" |

### Positioning Statement Template
```
"For [target customers] who are dissatisfied with [competitive alternative],
 [product] is a [market category] that provides [key value].
 Unlike [competitors], we [unique differentiator]."
```

**Critical insight**: Most category creation attempts fail. Use Head-to-Head or Big Fish Small Pond unless your approach is genuinely novel.

## Jobs-to-be-Done (JTBD) Framework

### Core Template
```
"When I [situation/trigger],
 I want to [motivation/job],
 so I can [expected outcome]."
```

### Three Job Layers
1. **Functional Job**: The practical task ("send invoices faster")
2. **Social Job**: How they want to be perceived ("look professional to clients")
3. **Emotional Job**: How they want to feel ("feel in control of cash flow")

### Messaging Application
```
Headline:     Address the functional job outcome
Subheadline:  Reference the emotional or social job
Body:         Focus on the job, not product features
```

## Value Proposition Canvas (Strategyzer)

### Customer Profile Side
```
Customer Jobs:  What tasks are they trying to accomplish?
Pains:          What frustrates them about current solutions?
Gains:          What outcomes would delight them?
```

### Value Map Side
```
Products/Services:  What you offer
Pain Relievers:     How you eliminate specific pains
Gain Creators:      How you create specific gains
```

### Copy Translation
```
Headline:     [Primary Gain Creator]
Subheadline:  [Primary Pain Reliever] + [for whom]
Bullet 1:     [Pain] → [Pain Reliever]
Bullet 2:     [Pain] → [Pain Reliever]
Bullet 3:     [Job] → [Gain Creator]
```

## Channel Prioritization: Bullseye Framework

From "Traction" by Weinberg & Mares.

### Three Rings
```
OUTER RING:   Brainstorm — list all 19 traction channels
MIDDLE RING:  Test — pick top 3 for cheap experiments ($100-500 each)
INNER RING:   Focus — double down on the one channel that works
```

### The 19 Traction Channels
```
1.  Viral Marketing          11. Trade Shows
2.  Public Relations          12. Offline Events
3.  Unconventional PR         13. Speaking Engagements
4.  Search Engine Marketing   14. Community Building
5.  Social/Display Ads        15. Targeting Blogs
6.  Offline Ads               16. Sales
7.  Search Engine Opt.        17. Affiliate Programs
8.  Content Marketing         18. Existing Platforms
9.  Email Marketing           19. Business Development
10. Engineering as Marketing
```

**Key stat**: 70% of startups find traction from a single channel. Optimize your core channel before expanding.

## ICE Scoring for Channel Selection

Score each channel 1-10 on three dimensions:

```
I = IMPACT:     How much will this move the key metric?
C = CONFIDENCE: How sure are you about the impact estimate?
E = EASE:       How easy/cheap is it to test?

ICE Score = (I + C + E) / 3
```

### Decision Template
```
| Channel          | Impact | Confidence | Ease | ICE Score | Action     |
|------------------|--------|------------|------|-----------|------------|
| Product Hunt     | 7      | 7          | 9    | 7.7       | Prioritize |
| Twitter/X thread | 6      | 7          | 9    | 7.3       | Prioritize |
| Email list       | 8      | 6          | 6    | 6.7       | Test       |
| Press outreach   | 7      | 3          | 4    | 4.7       | Backlog    |
| Reddit           | 5      | 5          | 8    | 6.0       | Test       |
| Paid ads         | 6      | 4          | 5    | 5.0       | Backlog    |
```

### Channel Fit by Product Type

| Product Type | Best Channels | Avoid |
|---|---|---|
| Developer tools | HN, Reddit, Twitter/X, GitHub | LinkedIn (unless B2B), Press |
| B2B SaaS | LinkedIn, Email, Press, Product Hunt | Reddit (usually), HN (unless technical) |
| B2C app | Product Hunt, Twitter/X, Reddit, Press | HN (unless technical angle) |
| Content tool | Product Hunt, Twitter/X, LinkedIn | HN (unless open-source) |
| Korean market | Naver Blog, KakaoTalk, 체험단, Naver Cafe | HN, Reddit, Product Hunt (unless dual-market) |
| Open-source | HN, GitHub, Reddit, Twitter/X | Press (too early), LinkedIn |

## Launch Timing Strategy

### Market Readiness Assessment (4 signals)
```
1. DEMAND SIGNAL:   Pre-registrations, waitlist, search volume trending up
2. TIMING NORMS:    Does launch align with buyer expectations?
3. COMPETITIVE GAP: Window before/after competitor moves?
4. SEASONAL FIT:    Does the quarter match buyer behavior?
```

### Four Timing Scenarios

| Scenario | Product Ready | Market Ready | Action |
|---|---|---|---|
| **Synergistic** | Yes | Yes | Launch immediately |
| **Internal Delay** | No | Yes | Accelerate internal prep |
| **Market Delay** | Yes | No | Build anticipation, soft launch |
| **Antagonistic** | No | No | Delay and reposition |

### Tactical Timing Rules
- Launching in the right quarter increases commercial success by up to 33%
- **Tuesday-Thursday** launches get the most coverage and engagement
- B2B: Avoid late Q4 (budget freeze) and early Q1 (planning mode)
- Never launch right before a major competitor's known announcement
- Check Product Hunt for conflicting major launches on your target day

### Launch Readiness Checklist
```
Product:  Core value works flawlessly. Onboarding is smooth.
Market:   Evidence of demand (feedback, waitlist, research).
Content:  Landing page, emails, social assets all produced and reviewed.
Team:     Everyone knows their launch day responsibilities.
Metrics:  Analytics, conversion tracking, dashboards configured.
Support:  Ready to handle increased volume on launch day.
```

## Competitive Analysis Framework

### Assess-Benchmark-Strategize

**Phase 1: Assess (Research)**
```
For each competitor (3-5), document:
- Positioning statement (how they describe themselves)
- Primary headline and subheadline
- Key benefits claimed (top 3-5)
- Social proof used (type and quantity)
- Pricing model and anchoring strategy
- CTA copy and funnel structure
- Channel focus (where they promote)
```

**Phase 2: Benchmark (Compare)**
```
| Element        | You      | Competitor A | Competitor B | Gap/Opportunity |
|----------------|----------|--------------|--------------|-----------------|
| Primary claim  |          |              |              |                 |
| Target segment |          |              |              |                 |
| Key benefit #1 |          |              |              |                 |
| Proof points   |          |              |              |                 |
| Price position |          |              |              |                 |
```

**Phase 3: Strategize (Act)**
```
Identify:
1. Messaging gaps — what pain points are competitors NOT addressing?
2. Underserved segments — who are competitors NOT targeting?
3. Proof gaps — where do competitors lack credible evidence?
4. Channel gaps — where are competitors NOT present?
```

### Differentiation Messaging Templates

**"Unlike" positioning:**
```
"Unlike [competitive alternative] that [limitation],
 [your product] [unique capability] so you can [outcome]."
```

**"Only" claim:**
```
"The only [category] that [unique differentiator]."
```

**"Before/With" contrast:**
```
"Before [Product]: [status quo pain]
 With [Product]:   [transformed outcome]"
```

**"We Believe" values-based:**
```
"We believe [philosophical stance competitors don't take].
 That's why we built [approach that embodies that belief]."
```
