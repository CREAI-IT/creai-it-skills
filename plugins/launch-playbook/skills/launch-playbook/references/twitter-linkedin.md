# Twitter/X & LinkedIn Launch Strategy

## Twitter/X

### Algorithm & Visibility (2025-2026)

Twitter/X prioritizes:
- **Engagement velocity** — replies, retweets, and likes in the first 30-60 minutes.
- **Thread completion** — longer threads that get read to the end signal quality.
- **Conversation** — tweets that generate replies rank higher than those that get only likes.
- **Account authority** — follower count, posting consistency, and engagement rate.
- **Media richness** — tweets with images/videos get 2-3x more engagement than text-only.

### Optimal Posting Times
- **Global/US audience**: 8-10 AM EST (weekdays), 12-1 PM EST
- **Tech/startup audience**: 9-11 AM PT (when SF/Bay Area is active)
- **Best days**: Tuesday, Wednesday, Thursday
- **Worst day**: Saturday

### Launch Thread Formula (10-15 tweets)

The launch thread is your primary Twitter asset. Structure:

**Tweet 1 — Hook (stop the scroll)**
```
I just launched [Product Name].

[One sentence: what it does + who it's for].

Here's the story of how it went from idea → live product 🧵
```

**Tweet 2 — Problem (empathy)**
```
The problem:

[Describe the pain point in 2-3 bullet points]
[Be specific — "I was spending 4 hours/week on X" not "X was hard"]
```

**Tweet 3 — Failed alternatives**
```
I tried everything:
• [Alternative 1] — too expensive
• [Alternative 2] — too complex
• [Alternative 3] — didn't solve the core problem

Nothing worked.
```

**Tweet 4 — Insight / Aha moment**
```
Then I realized:

[The insight that led to building the product]
[This should be a non-obvious realization]
```

**Tweet 5 — Solution introduction**
```
So I built [Product Name].

[One-line description of what it does]

[Screenshot or GIF of the product]
```

**Tweet 6-8 — Key features (one per tweet)**
```
Feature [N]: [Benefit-driven name]

[1-2 sentences explaining what it does and why it matters]

[Screenshot showing the feature]
```

**Tweet 9 — Social proof (if available)**
```
Early users are already seeing results:

"[Testimonial quote]" — @username

[Or: "X users in the first Y days" / "Z% improvement in [metric]"]
```

**Tweet 10 — Behind the scenes**
```
Building this took [timeframe].

Some numbers:
• [Lines of code / hours worked / iterations]
• [Stack: built with X, Y, Z]
• [One interesting technical challenge]
```

**Tweet 11 — The ask**
```
If this sounds useful, I'd love for you to try it:

[Product URL]

🔥 Special launch offer: [discount/trial/bonus]

And if you know someone who'd benefit, a RT would mean the world 🙏
```

**Tweet 12 — Engagement hook (final)**
```
What's the biggest challenge you face with [problem area]?

Reply below — I'm reading every response and it helps me build better.
```

### Pre-Launch Teaser Tweets (post T-7 to T-1)

**Teaser 1 (T-7): Problem awareness**
```
Unpopular opinion: [contrarian take about the problem space]

I've been working on something to fix this.

More soon 👀
```

**Teaser 2 (T-4): Building in public**
```
Sneak peek at what I've been building for the past [timeframe]:

[Screenshot/GIF]

Launching [day]. Excited to share more.
```

**Teaser 3 (T-1): Countdown**
```
Tomorrow I'm launching [Product Name] on @ProductHunt.

[One sentence: what it does]

Drop a 🔥 if you want me to share the link when it's live.
```

### Post-Launch Engagement Tweets

**Results sharing (T+1):**
```
[Product Name] launch results:

📈 [X] upvotes on Product Hunt
💬 [X] comments
👥 [X] new signups
🏆 #[N] Product of the Day

Thank you to everyone who supported!

Here's what I learned 🧵
```

**User feedback showcase (T+2-3):**
```
My favorite piece of feedback so far:

"[User quote]"

This is exactly why I built [Product].

[Screenshot of feedback]
```

**Lessons learned (T+3-5):**
```
5 things I'd do differently if I launched again:

1. [Lesson]
2. [Lesson]
3. [Lesson]
4. [Lesson]
5. [Lesson]
```

### Hashtag Strategy
Use 3-5 hashtags max. Mix of:
- **Category**: #SaaS, #NoCode, #DevTools, #AI
- **Action**: #buildinpublic, #indiehacker, #launch
- **Platform**: #ProductHunt (on launch day only)

Don't use generic hashtags (#startup, #tech, #innovation) — too noisy, no targeting value.

## LinkedIn

### Algorithm & Visibility (2025-2026)

LinkedIn prioritizes:
- **Dwell time** — how long people spend reading your post.
- **Early engagement** — comments in the first 60-90 minutes.
- **Meaningful comments** — comments over 10 words count more than reactions.
- **Consistency** — regular posters get more reach.
- **Native content** — LinkedIn penalizes external links in posts. Put links in comments.

### Launch Post Formula

**Format**: Hook → Story → Value → CTA

```
[Hook line — bold statement, question, or contrarian take]

[2-3 line break for visual spacing]

Here's the story:

[Background — why you built this. 3-4 sentences, personal and specific.]

[What the product does — 2-3 sentences, benefit-focused.]

[Results or social proof — numbers, testimonials, early traction.]

[What you learned — 1-2 insights from the process.]

—

[CTA — "Link in comments" or "DM me for early access"]

[3-5 relevant hashtags]
```

**Example structure:**
```
I quit my job 6 months ago to solve a problem I couldn't stop thinking about.

Today, I'm launching it publicly.

[Story of the problem]
[What the product does]
[Early results]
[What's next]

If you're dealing with [problem], I'd love your feedback.

Link in the first comment 👇

#buildinpublic #SaaS #launch
```

### LinkedIn Rules
- **Put links in the first comment**, not the post body. Posts with external links get 40-50% less reach.
- **No emojis in every line** — LinkedIn is professional. Use sparingly.
- **Hook must be strong** — LinkedIn only shows first 2-3 lines before "see more."
- **Tag people** who were involved (co-founders, early supporters, mentors). Tagged people's networks see the post.
- **Post between 7-9 AM** in your primary audience's timezone (Tuesday-Thursday best).
- **Respond to every comment** within the first 2 hours.

### Founder Story Post (alternative to launch post)

Use this format for a more personal, narrative-driven post:

```
[Month] ago, I had a conversation that changed everything.

[The conversation or moment that sparked the idea]

[The problem — specific, relatable]

[The failed attempts — what you tried first]

[The insight — what clicked]

[What you built — 2-3 sentences]

[Where you are now — traction, users, learning]

The lesson: [One takeaway]

—

[Product name] is live. Link in comments.
```

### B2B vs B2C Differences

| Element | B2B | B2C |
|---|---|---|
| Platform priority | LinkedIn > Twitter | Twitter > LinkedIn |
| Tone | Professional, data-driven | Casual, personality-driven |
| Social proof | Company logos, revenue metrics | User count, testimonials |
| CTA | "Book a demo" / "See case study" | "Try it free" / "Sign up" |
| Content type | Case studies, ROI data | Product screenshots, GIFs, memes |
| Hashtags | Industry-specific | Broad community (#buildinpublic) |
| Best posting time | 7-9 AM business timezone | 9-11 AM, 12-1 PM user timezone |

### Carousel Posts (LinkedIn)

LinkedIn carousels (PDF uploads) get 3-5x more engagement than text posts.

**Launch carousel structure (8-10 slides):**
1. Title slide: Bold headline + product name
2. Problem slide: "You're probably dealing with..."
3. Agitate: Cost of the problem (time, money, frustration)
4. Solution: "[Product] fixes this by..."
5-7. Feature slides: One feature per slide, benefit-focused
8. Social proof: Testimonials or metrics
9. Offer: "Special launch pricing" or "Free trial"
10. CTA: "Link in comments" + contact info

**Design rules:**
- Use 1080x1350px (4:5 ratio)
- Large text (minimum 24pt)
- Consistent brand colors
- One idea per slide
- Number slides ("3/10") so people know there's more
