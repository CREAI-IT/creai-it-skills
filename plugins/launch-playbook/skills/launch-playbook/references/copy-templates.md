# Copy Templates & Conversion Formulas

## Landing Page Copy Frameworks

### AIDA (Attention - Interest - Desire - Action)
```
ATTENTION: Bold headline that stops the scroll
INTEREST:  Subheadline + supporting copy that satisfies search intent
DESIRE:    Benefits, social proof, testimonials, trust signals
ACTION:    Clear, single CTA
```

**Template:**
```
Headline:    "Stop [pain] — Start [desired outcome]"
Subheadline: "[Product] helps [audience] achieve [outcome] with [mechanism]"
Body:        3 benefits (not features), each with proof point
CTA:         "Get [specific outcome] Now"
```

**Best for**: Product pages, paid ad landing pages, simple offers.

### PAS (Problem - Agitate - Solution)
The highest-converting framework for pain-driven products.

```
PROBLEM:   Name the exact pain the reader feels right now
AGITATE:   Twist the knife — show the cost of inaction
SOLUTION:  Present your product as the clear path out
```

**Agitation examples:**
- "Every bland headline is money walking out the door."
- "You're paying $5-15 per click for tire-kickers who'll never buy."
- "Every missed deadline and confused team member is costing you time and money."

**Best for**: Email copy, landing pages for pain-driven products, B2B SaaS.

### Before-After-Bridge (BAB)
More positive framing than PAS. Focuses on transformation.

```
BEFORE:  "Right now, you're [struggling with X]. It takes [time/effort],
          and you're still [not getting result]."
AFTER:   "Imagine [desired state]. [Specific metric improvement].
          [Emotional benefit]."
BRIDGE:  "[Product] makes this possible by [mechanism]. Here's how..."
```

**Best for**: Homepage hero sections, pitch decks, aspirational products.

### StoryBrand (Donald Miller's 7-Part Framework)
Positions customer as hero, brand as guide.

```
1. CHARACTER:      Your customer wants something
2. PROBLEM:        External, internal, and philosophical problem
3. GUIDE:          Your brand (with empathy + authority)
4. PLAN:           Simple 3-step plan
5. CALL TO ACTION: Direct + transitional CTA
6. FAILURE:        What they risk if they don't act
7. SUCCESS:        The positive transformation
```

**BrandScript template:**
```
"[Customer type] who struggle with [external problem] feel [internal problem]
because [philosophical problem].

[Brand] understands [empathy statement] and has [authority signal].

Here's the simple plan:
  Step 1: [Easy first action]
  Step 2: [Implementation]
  Step 3: [Achieve desired result]

[Direct CTA]: Start [outcome] today.
[Transitional CTA]: Download our free [resource].

Without this, you risk [failure state].
With it, you'll [success state]."
```

### Framework Selection Matrix

| Scenario | Framework | Why |
|---|---|---|
| Pain-driven buyer | PAS | Agitation drives urgency |
| Aspirational buyer | BAB | Transformation narrative |
| Complex product | StoryBrand | Narrative clarity |
| Simple offer / ads | AIDA | Fast, linear persuasion |

## Headline Formulas

### Formula 1: Outcome + Timeframe + Objection Override
```
"[Achieve X] in [Timeframe] — Even If [Common Objection]"

Example: "Build a Profitable SaaS in 90 Days — Even If You Can't Code"
```

### Formula 2: How-To Without Pain
```
"How to [Desired Outcome] Without [Pain Point]"

Example: "How to Get 1,000 Subscribers Without Spending a Dime on Ads"
```

### Formula 3: Number + Adjective + Outcome
```
"[Number] [Adjective] Ways to [Outcome] That [Proof]"

Example: "7 Proven Ways to Double Your Landing Page Conversions"
```

### Formula 4: Question + Challenge
```
"Are You Making These [Number] [Topic] Mistakes?"

Example: "Are You Making These 5 Pricing Page Mistakes?"
```

### Formula 5: Curiosity Gap
```
"The [Unexpected Adjective] [Method] That [Impressive Result]"

Example: "The Counterintuitive Email Strategy That Generated $2.3M"
```

### Formula 6: Direct Value Proposition
```
"[Product Type] That [Does Primary Job] So You Can [Ultimate Benefit]"

Example: "AI Assistant That Drafts Landing Pages So You Can Launch in Hours"
```

### Formula 7: Social Proof Headline
```
"[Number] [Customer Type] Use [Product] to [Outcome]"

Example: "14,200 Marketing Teams Use ConvertKit to Grow Their Audience"
```

### Headline Rules
- Optimal length: 8-12 words
- Odd numbers outperform even by 20%
- Use "you" not "we/us"
- Specific numbers beat vague claims ("14,200 teams" > "thousands of teams")
- Power phrases: "will make you", "simple way to", "that actually work"

## Subheadline Patterns

Optimal length: 12-35 words.

**Template A: Mechanism + Audience**
```
"[Product type] that [mechanism] for [specific audience]."
```

**Template B: Objection Handler**
```
"No [common objection]. No [common objection]. Just [desired outcome]."
```

**Template C: Proof Point**
```
"Built by [credibility]. Trusted by [social proof metric]."
```

**Template D: Benefit Stack**
```
"[Benefit 1]. [Benefit 2]. [Benefit 3]."
```

**Rule**: The headline makes the promise; the subheadline makes it credible.

## Feature-to-Benefit Translation

### The "So What?" Method
```
1. Write down the feature
2. Ask "So what?" → functional advantage
3. Ask "So what?" again → emotional/outcome benefit (USE THIS)
```

### Translation Table

| Feature | Advantage | Benefit (Use This) |
|---|---|---|
| "256-bit encryption" | "Data is protected" | "Sleep soundly knowing customer data can never be breached" |
| "AI-powered analytics" | "Patterns detected automatically" | "Spot revenue opportunities your competitors miss" |
| "One-click integration" | "Works with existing tools" | "Get started in 5 minutes, not 5 weeks" |
| "Real-time collaboration" | "Team sees changes instantly" | "Kill the version-control chaos that delays every project" |
| "24/7 live support" | "Help whenever needed" | "Never feel stuck or alone when something goes wrong" |

### Copy Pattern
```
"[Feature name]: [One-line benefit, not description]"

BAD:  "AI Analytics: Machine learning algorithms process your data"
GOOD: "AI Analytics: Spot the revenue opportunities your competitors miss"
```

## CTA Copy Patterns

### What Converts
```
"Get Your Free Demo"          (specific)     ✓
"Start Your 14-Day Trial"     (specific)     ✓
"See [Product] in Action"     (curiosity)    ✓
"Join 14,200+ Teams"          (social proof) ✓
"Create My First [Outcome]"   (ownership)    ✓
"Show Me How It Works"        (low risk)     ✓
```

### What Doesn't Convert
```
"Submit"        (vague)              ✗
"Click Here"    (meaningless)        ✗
"Learn More"    (weak)               ✗
"Sign Up"       (generic)            ✗
"Buy Now"       (too aggressive)     ✗
```

### CTA Hierarchy
```
PRIMARY (Direct):       "Start My Free Trial"
TRANSITIONAL (Soft):    "Download the Free Guide"
REPEAT:                 Same CTA at top, middle, and bottom
```

### CTA Stats (2025-2026)
- Personalized CTAs convert 202% better than generic
- CTAs at both top and bottom: +25% conversion
- First person ("Start **my** trial") outperforms second person ("Start **your** trial")
- Single CTA per email increases clicks by 371%
- Minimum button size: 44x44 pixels for mobile

## Social Proof Patterns

### Pattern 1: Specificity Proof
```
"Used by 14,200 teams across 38 countries"
"4.9/5 stars from 3,200+ reviews"
```
Rule: Specific numbers beat round numbers. "14,200" > "14,000".

### Pattern 2: Outcome-Framed Testimonials
```
"[Name], [Title] at [Company]
 '[Specific outcome] [in timeframe]. [Emotional reaction].'"
```

### Pattern 3: Before/After Stats
```
"Before [Product]: 2% conversion rate, 40 hours/week manual
 After [Product]:  8.5% conversion rate, 4 hours/week automated"
```

### Pattern 4: Logo Bar + Metric
```
"Trusted by [Logo] [Logo] [Logo] and 2,000+ others"
```

### Placement
Social proof should appear:
- Above the fold (mini version: logo bar or stat)
- After the hero section (full testimonials)
- Near every CTA

## Urgency & Scarcity (Ethical Only)

41% of consumers report "fake deadline fatigue." Only use truthful urgency.

### Genuine Deadline
```
"Founding member pricing ends [specific date]"
"Early-bird rate available until we reach 500 users"
```

### Capacity-Based Scarcity
```
"We onboard 10 new clients per month to ensure quality"
"Only 3 implementation slots remaining for Q2"
```

### Cost of Delay
```
"Every week without [solution], you're leaving $X on the table"
"Your competitors are already [doing the thing you enable]"
```

### What to Avoid
```
✗ Fake countdown timers that reset on page refresh
✗ "Only 2 left!" when there's unlimited supply
✗ Perpetual "sale ending soon" that never ends
```

## Landing Page Section Structure

### Recommended Full-Page Order
```
Section 1:  HERO — Headline + Subheadline + CTA + Visual + Mini Social Proof
Section 2:  SOCIAL PROOF BAR — Logo bar + aggregate stat
Section 3:  PROBLEM STATEMENT — PAS: Problem + Agitate
Section 4:  SOLUTION — 3-step plan or product walkthrough
Section 5:  FEATURES → BENEFITS — 3-5 capabilities as outcomes
Section 6:  SOCIAL PROOF (DEEP) — 2-3 detailed testimonials
Section 7:  COMPARISON (optional) — vs. alternatives table
Section 8:  PRICING (if applicable) — clear tiers, highlighted recommended
Section 9:  FAQ — 5-8 questions handling buying objections
Section 10: FINAL CTA — repeat hero headline + CTA + urgency element
```

CTA placement: minimum 3 times (Section 1, 6, and 10).

### Above-the-Fold Formula
```
1. HEADLINE:     Promise the primary outcome (6-8 words)
2. SUBHEADLINE:  Explain the mechanism + for whom (12-35 words)
3. CTA:          Single, clear, benefit-driven button
4. HERO VISUAL:  Product screenshot, demo GIF, or aspirational image
5. MINI PROOF:   Logo bar or single stat (optional)
```

Rule: If someone can't explain your offer after 5 seconds of reading the fold, you have a clarity problem.

### FAQ Section Formula
5-8 questions max. Address the 5 universal objection categories:

| Category | Example |
|---|---|
| Price/Value | "Is it worth the investment?" |
| Risk | "What if it doesn't work for me?" |
| Effort | "How long does setup take?" |
| Trust | "Who else uses this?" |
| Timing | "Can I cancel anytime?" |

**FAQ answer formula:**
```
Q: [Objection as customer would say it]
A: [Empathy]. [Direct answer]. [Proof]. [Micro-CTA]
```

## Korean Copy Adaptation Notes

When producing Korean versions of copy:

- **Don't translate literally.** Rewrite for Korean sentence patterns and cultural context.
- **Korean headlines are shorter.** 6-10 words vs. 8-12 in English.
- **Korean descriptions are shorter.** 80-120 characters vs. 150-160 in English.
- **Korean CTA patterns differ.** Use `~하기` (verb + 하기) format: "시작하기", "체험하기", "확인하기".
- **Honorific level matters.** Use 해요체 for B2C, 합니다체 for B2B.
- **Numbers and social proof translate well.** "14,200개 팀이 사용 중" works directly.
- **Emotional tone differs.** Korean copy is often more empathetic and less aggressive than English.
