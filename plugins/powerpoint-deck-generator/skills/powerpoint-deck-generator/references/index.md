# Design Reference — Module Index

This is a modular design reference system. **Do not read all modules.** Read `index.md` (this file) first, then load only the modules relevant to your current task phase.

## Module Map

| Module | File | Read when... |
|--------|------|-------------|
| **Narrative & Structure** | `narrative-structure.md` | Planning the deck: writing action titles, structuring the SCR arc, applying the ghost deck process |
| **Typography** | `typography.md` | Choosing fonts, setting size hierarchy, applying typographic techniques |
| **Color & Palette** | `color-palette.md` | Building the color palette, choosing dark/light theme, applying accent strategy |
| **Layout & Composition** | `layout-composition.md` | Designing slide layouts, managing density, sizing cards/containers, choosing body patterns |
| **Texture & Atmosphere** | `texture-atmosphere.md` | Adding depth to slides: CSS noise/grain, radial glows, grid patterns, decorative elements |
| **Slide Type Patterns** | `slide-types.md` | Building a specific slide type: title, data/metric, comparison, process, closing |
| **Print & Export** | `print-export.md` | Preparing for PDF export: `backdrop-filter` fixes, flex centering, browser print quirks |
| **Quality & Anti-Patterns** | `quality-checklist.md` | Final review: checklist, source citations, anti-pattern avoidance |

## Phase → Module Loading Guide

### Phase 1: Planning (`plan.md`)
1. **Always read**: `narrative-structure.md` — for action titles, SCR framework, storyline test
2. **Skim**: `slide-types.md` — to pick the right slide type per content
3. **Skim**: `typography.md` + `color-palette.md` — to commit to a design direction

### Phase 2: Building (slides + styles.css)
1. **Always read**: `layout-composition.md` — for density rules, card sizing, spatial patterns
2. **Always read**: `typography.md` — for font hierarchy and size enforcement
3. **Read per need**: `texture-atmosphere.md` — when setting up slide backgrounds
4. **Read per need**: `slide-types.md` — when building each specific slide type
5. **Read per need**: `color-palette.md` — when customizing `styles.css`

### Phase 3: Review & Export
1. **Always read**: `quality-checklist.md` — run every check before delivering
2. **Always read**: `print-export.md` — before any PDF export (browser print or Playwright)

## Design Philosophy

Every deck must feel like it was designed by a human with strong opinions — not generated by a template engine. The difference between a forgettable deck and a memorable one is **intentionality**: every font choice, color decision, and layout should serve the content and feel considered.

**The goal**: Someone sees the deck and thinks "who designed this?" — not "oh, another AI deck."
