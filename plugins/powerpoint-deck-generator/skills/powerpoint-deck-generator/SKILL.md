# PowerPoint Deck Generator

Generate consulting-quality presentation decks as HTML/CSS/JS files with 16:9 aspect ratio. Each slide is a separate HTML file, wired together via an `index.html` viewer with arrow-key navigation and PDF export.

Every deck must look like it was crafted by a senior presentation designer — not generated by AI.

<HARD-GATE>
Do NOT generate any slide HTML files until the user has explicitly approved the plan.md. No exceptions.
</HARD-GATE>

---

## Non-Negotiable Rules

These apply to EVERY slide. Violations are the top causes of rework.

<CRITICAL>
**#1 — Sparse slides**: Fill 80%+ of the canvas. Every non-title slide needs: overline + action title + body text + 4+ data elements + supporting section. Title + one stat + whitespace = draft, not a slide.

**#2 — Font sizes too small**: Body/card/description text starts at **24px**, never below **22px**. Never 16-20px for readable content. If it doesn't fit, cut words or split the slide.

**#3 — Export artifacts (BANNED CSS)**: NEVER use `text-shadow` (blur > 0), `box-shadow` with blur on elements < 100px (dots, nodes, badges), or `backdrop-filter` in slide CSS. Creates rectangular "drag mark" artifacts in Playwright PDF. Use brighter colors, borders, and solid `rgba()` backgrounds instead.

**#4 — Stretched cards**: NEVER use `flex: 1` on card containers in column layouts or `grid-template-rows: 1fr`. Cards must be content-sized (`auto` rows).
</CRITICAL>

→ Full rules: `references/quality-checklist.md` | `references/print-export.md` | `references/layout-composition.md`

## Phase 1: Plan

### Step 1: Understand the Deck

Clarify: topic/purpose, audience, slide count, visual tone, specific content per slide.

### Step 2: Scaffold Folder + Write plan.md

#### 2a: Create the deck folder
Descriptive kebab-case name (e.g., `ai-startup-pitch/`). Check if it already exists.

#### 2b: Narrative Architecture

Structure the argument using **SCR** (Situation → Complication → Resolution):
- **Situation** (10–15%): Uncontroversial baseline
- **Complication** (15–20%): Why action is needed
- **Resolution** (60–70%): Your recommendation + evidence

Write all **action titles first** (ghost deck): complete-sentence headlines stating conclusions, not topic labels. Max 2 lines, 15 words, must contain a verb. Apply the storyline test — titles in sequence must form a coherent argument.

→ Full methodology: `references/narrative-structure.md`

#### 2c: Design Direction

Commit to a bold aesthetic: specific tone, distinctive font pair (NEVER Inter/Roboto/DM Sans/Poppins), 3-4 colors with functional meaning, spatial signature, texture, one differentiator.

→ `references/typography.md` | `references/color-palette.md`

#### 2d: Write plan.md

Read the template, then write to `<deck-folder>/plan.md`:
- `~/.claude/skills/powerpoint-deck-generator/assets/plan-template.md`

### Step 3: User Review

Present the plan and ask for feedback on slide structure, action titles, content, and design direction. Iterate until approved.

<HARD-GATE>
Do NOT proceed to Phase 2 until user explicitly approves. "go", "looks good", "build it", "진행해", "좋아 만들어" = approval.
</HARD-GATE>

## Phase 2: Build

### Step 4: Design Theme + Build Slides

Only after user approval.

#### 4a: Read Assets + References

**Assets** (templates):
- `~/.claude/skills/powerpoint-deck-generator/assets/viewer-template.html` → `index.html`
- `~/.claude/skills/powerpoint-deck-generator/assets/slide-template.html` → `slide-N.html`
- `~/.claude/skills/powerpoint-deck-generator/assets/base-styles.css` → `styles.css`

**References** — read the index, then load per phase:
- `~/.claude/skills/powerpoint-deck-generator/references/index.md` — routing table

#### 4b: Create styles.css

Copy from `assets/base-styles.css`, customize: `--accent`/`--bg`/`--text` CSS variables, Google Font imports, deck-specific utilities.

#### 4c: Create slide files

**`slide-N.html`** — Based on `assets/slide-template.html`. Replace `{{SLIDE_TITLE}}`, `{{DECK_TITLE}}`, `{{SLIDE_SPECIFIC_STYLES}}`, `{{SLIDE_CONTENT}}`, `{{SPEAKER_NOTES}}`. Follow plan.md exactly. Link to `styles.css`. Standalone-viewable. Add speaker notes via HTML comment before closing `</body>`: `<!-- notes: Talking points for this slide -->`.

**Per-slide density check**: (1) body text below title, (2) 4+ data elements, (3) supporting section at bottom. Missing any → add content before proceeding.

**`export_pdf.py`** — Copy: `cp ~/.claude/skills/powerpoint-deck-generator/assets/export_pdf.py <deck-folder>/`

**`index.html`** — Based on `assets/viewer-template.html`. Replace `{{DECK_TITLE}}`, `{{DECK_STYLES}}` → `<link rel="stylesheet" href="styles.css">` (BEFORE inline `<style>`), `{{SLIDE_FILES}}` → comma-separated slide list.

### Step 5: Serve and Open

```bash
# Run with run_in_background=true:
python3 -m http.server 8731
# Then separately:
open http://localhost:8731
```

### Step 6: Export PDF (mandatory)

```bash
cd <deck-folder> && python3 export_pdf.py
```

Auto-starts its own server. Exports through the viewer for pixel-perfect CSS fidelity. Every completed deck MUST have a `deck-export.pdf`.

→ Export details: `references/print-export.md`

## Quick Reference: Font Size Hierarchy (1920×1080)

Always start at the **upper end** of ranges. When in doubt, go larger.

| Tier | Element | Size | Weight |
|------|---------|------|--------|
| **T1** | Action title | 38–44px | 600–700 | Same on every slide. Left-aligned. |
| **T2** | Body/subheading below title | 24–28px | 400–500 |
| **T3** | Card content, descriptions, lists | 22–28px (default 24px) | 400 |
| **Hero** | Key metric / hero number | 64–120px | 700–900 |
| **T4** | Labels, overlines, badges | 14–18px | 600–700 | Uppercase, tracked. |
| **T5** | Source citations only | 12–14px | 400 | The ONLY text below 14px. |

**Hard floor**: Nothing below 14px except sources. Nothing below 22px if meant to be read. Max 2 font families, 3 sizes per slide.

→ Full hierarchy + techniques: `references/typography.md`

## Quick Reference: Slide Elements (every non-title slide)

| # | Element | Spec |
|---|---------|------|
| 1 | **Overline** | Section label, uppercase, T4 |
| 2 | **Action title** | Complete sentence stating a conclusion, T1 |
| 3 | **Body text** | 1-2 sentences of context, T2 |
| 4 | **Primary content** | Cards, grids, columns, hero metrics — 4+ data elements |
| 5 | **Supporting section** | Callout bar, impact row, comparison strip, or key quote |

→ Density targets per slide type: `references/layout-composition.md` | `references/slide-types.md`
